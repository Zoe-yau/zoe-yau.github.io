   All three of the graphs show the average runtime of trials that I ran on my Bubble Sort, Insertion Sort, and Selection Sort algorithms. Bubble sort works by repeatedly swapping elements until they are in least to greatest order, insertion sort similarly swaps the elements except it compares each element one at a time, and selection sort finds the smallest number in the list and moves it to the front to order them. For each algorithm I ran 5 trials of text files consisting of 1 word, 10 words, 100 words, 1000 words, 10000 words, and 100,000 words. Each text file also had three types - ordered, reverse ordered, and randomized. The x-axis of the graphs represents the number of words that were in each file, and the y-axis represents the time averages of each trial in milliseconds. The three lines on each graph represent each of the file types (ordered, reversed, random). The graphs all grow exponentially due to the time increasing as the number of words in each file is multiplied by 10, so when the lines reach 10,000 words the lines dramatically increase.
   Bubble sort was overall the fastest of the three while selection and insertion sorts had similar results. Amongst all three file types the files that were ordered and only contained one word on average took the longest. For each sort the results for files with 1 and 10 words were fairly similar taking around 4 - 5 milliseconds. Where insertion and selection sort differ from bubble sort is around 100 words insertion and selection sort would hit a sharp increase while bubble sort has a more gradual increase. Overall the data collected was able to effectively show how fast each algorithm works with various files.

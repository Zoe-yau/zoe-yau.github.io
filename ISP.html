   A game where the player will get six anonymous contestants as potential love interests. They will be hidden to the player until the end of the game. The contests are asked a list of questions, and the player will pick whose answer they like best. At the end the answers are tallied up and the most chosen contestant is the one for the player.

   My friends and I really enjoyed watching dating shows like the bachelorette, so we wanted to make a game for us to play with a similar concept. Something that would be fun and enjoyable and easy to understand.

  • The update single stack function is how each answer choice changes for each question

  • Choice button pressed function tallies up the answers by taking into account which answer was chosen

  • The calculate personality result function calculates the result by taking each picked answer and storing the choice in a variable. It then sorts through the chosen answers to figure out which had the highest chosen frequency

   The overall design and the idea of the game went well. However, we origianlly wanted to have more contestants but because of the time limit we were only able to draw 6. Also in the game initially we wanted to have only 3 contestants to be options but we eventually changed that to all because it gave the game a variety of choices making it more fun. Overall, we had a really great time getting to collaborate and create an awesome game!
